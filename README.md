# Uber Performance Analysis

## Overview
This Power BI project provides comprehensive analysis and visualization of Uber performance metrics. The report enables stakeholders to track key performance indicators, identify trends, and make data-driven decisions.

## Project Structure

### Files & Folders
- **Uber Performance_ Analysis.pbip** - Main Power BI project file
- **Uber Performance_ Analysis.Report/** - Report definition and visuals
- **Uber Performance_ Analysis.SemanticModel/** - Data model and relationships

### Report Pages
The project includes multiple interactive pages with visualizations:
- Page 1 (`2b6eaec7276e565c943b`) - Primary analysis dashboard
- Page 2 (`c76e8b53c96714e14148`) - Secondary metrics and insights
- Page 3 (`d24ad28a1b60c961d598`) - Additional performance analysis

### Data Model
The semantic model includes the following tables:
- **Trip Details** - Core transaction-level data
- **Location Table** - Geographic and location information
- **Calendar Table** - Time-based analysis support
- **Dynamic Measure** - Calculated metrics and KPIs

## Key Features
- Multiple interactive dashboards
- Time-based analysis with calendar tables
- Location-based performance tracking
- Dynamic measures for flexible analysis
- Bookmarks for saved view states

## How to Use
1. Open `Uber Performance_ Analysis.pbip` in Power BI Desktop
2. Navigate between report pages using the page tabs
3. Use filters and slicers to explore specific metrics
4. Click bookmarks to quickly return to saved views
5. Refresh data connections as needed for updated analysis

## Data Sources
Review the semantic model connections to understand data sources and table relationships in the `definition/` folders.

## Notes
- All data transformations are configured in the semantic model
- Report themes are managed in the BaseThemes configuration
- Bookmarks are available for quick navigation between key views
